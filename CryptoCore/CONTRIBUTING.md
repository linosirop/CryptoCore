# Contributing

## Как предложить изменения
1. Создай ветку от `main`
2. Делай небольшие коммиты с понятными сообщениями
3. Добавляй/обновляй тесты
4. Обновляй документацию при изменении публичного API или CLI

## Стиль кода
- C++17/20 (как у тебя в проекте)
- Явные проверки ошибок
- Безопасная обработка буферов и размеров

## Pull Request checklist
- [ ] Тесты проходят
- [ ] Документация обновлена
- [ ] Нет вывода секретов
- [ ] Проверки входных данных добавлены/обновлены
